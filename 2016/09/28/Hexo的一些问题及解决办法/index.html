Hexo的一些问题及解决办法

发表于 2016-09-28   |  

问题1：在用Hexo搭博客在执行hexo g的时候，会出现错误：
ERROR Process failed: layout/.DS_Store TypeError: Cannot read property 'compile' of undefined 或者ERROR Process failed: layout/_partial/.DS_Store TypeError: Cannot read property 'compile' of undefined

首先得知道.DS_Store是 OS X 系统生成的文件，它是用来存储这个文件夹的显示属性的 。Finder 用它记录客户定制文件夹显示方式的一种元数据文件（metadata）用于控制一个文件夹的显示属性（图标位置、视图设置等）。 但是因为在 OS X 系统中 Finder 自动隐藏它们，所以在 Mac 下交互使用这些文件是没有任何问题的。.DS_Store几乎存在于每个文件当中，删除它会丢失它记录的那些信息。和别人交换文件（或做网页需要上传的时候）应该把.DS_Store文件删除比较妥当，因为里面包含了一些不一定希望别人看见的信息（尤其是网站，通过.DS_Store可以知道这个目录里面所有文件的清单）。这里有代码可以禁止生成及恢复生成：

1.禁止 .DS_Store 生成：
打开 Terminal 输入下面的命令并按下 enter 键运行，然后必须注销Mac并重新登陆系统才能生效。     defaults write com.apple.desktopservices DSDontWriteNetworkStores -bool TRUE

2.允许 .DS_Store 生成：
（也是需要在 Terminal 下操作，注销重启后生效）
defaults delete com.apple.desktopservices DSDontWriteNetworkStores

这个对我们部署没有任何帮助，只是避免以后再出现类似的问题。现在要删除.DS_Store这个文件，分别在目录layout和layout/_partial下，这里的目录是Hexo的主题主目录前提下，比如我的是next主题，本来已经在hexo主目录下了：

1. 删除 layout/.DS_Store:
cd     themes/next/layout
rm －fr .DS_Store         ＃我的执行完了感觉没什么变化，终端里没有代码提示
2. 删除 layout/_partial/.DS_Store:
cd themes/next/layout/_partial/.DS_Store
rm －fr .DS_Store

问题2：在执行hexo d的时候出现fatal: Not a git repository (or any of the parent directories): .git的问题，导致没法发表博文。

我是删除 hexo 根目录下的文件.deploy_git，重新hexo deploy，然后就好了。

问题3：在执行hexo d的时候出现YAMLException: end of the stream or a document separator is expected at line x, column y

出现这种情况，一般都是缺少空格，在:冒号之后要有空格，检查x行y列附近的冒号，其之后是否有空格。
