Python 安装图像处理库 PIL

Veröffentlicht am 2016-09-21   |  

在 Python 中，安装第三方模块，是通过 setuptools 这个工具完成的。Python 有两个封装了 setuptools 的包管理工具：easy_install和pip。目前官方推荐使用pip。

PIL(Python Imaging Library)是Python一个强大方便的图像处理库，名气也比较大。一般来说第三方库 pip 直接安装即可，所以安装 Python Imaging Library 的命令就是：
pip install PIL，但是出现了Could not find a version that satisfies the requirement PIL (from versions: ) No matching distribution found for PIL错误，百度才知道 PIL 不兼容 setuptools ，而且报告的问题太多，更新太慢。

Pillow 是 PIL 的一个派生分支，但如今已经发展成为比 PIL 本身更具活力的图像处理库。用pip install Pillow命令安装，安装完成，可以使用from PIL import Image就可以使用库。

这里一般会碰到一个问题，比如我想打开/Users/用户名/Downloads目录下的名为70.jpg图片：

im = Image.open('/Users/用户名/Downloads/70.jpg ')
Traceback (most recent call last):
      File "<stdin>", line 1, in <module>
     File "/Library/Python/2.7/site-packages/PIL/Image.py", line 2280, in open
            fp = builtins.open(filename, "rb")
IOError: [Errno 2] No such file or directory: '/Users/用户名/Downloads/70.jpg '

后来发现都是在当前目录操作的，可以用：

import os
print(os.getcwd())

可以查看你当前目录，图片文件放在当前目录才可以，当然也可以更换当前目录，修改的命令是：
os.chdir('/Users/wking/Downloads')

对 PIL 更多使用可以翻手册，中文版；英文版。

参考：

  1. python Pillow 处理图片
  2. [ller必读] LoveLive! 必备技能之 Python Pillow 自动处理截图
  3. python-pillow库PIL.Image.transform()方法详解

Mac下tarball方式安装MySQL

Veröffentlicht am 2016-09-20   |  

今天安装MySQL，直接下载dmg包安装很无趣，然后鬼使神差的又玩起了tarball。如下：

系统环境: OS X EI Capitan 10.11.3
MySQL版本: 5.6.33 (mysql-5.6.33-osx10.11-x86_64.tar.gz)
MySQL下载: http://dev.mysql.com/downloads/mysql/5.6.html

下载mysql-5.6.33-osx10.11-x86_64.tar.gz文件后，照样先进入下载目录，解压：

cd /Users/用户名/Downloads
tar zxvf mysql-5.6.33-osx10.11-x86_64.tar.gz

这里选择默认安装目录/usr/local：

可以先把 mysql-5.6.33-osx10.11-x86_64 重命名为 mysql ，这样简洁方便。把重命名后的 mysql 文件夹二进制包到安装目录    
sudo mv mysql /usr/local

# 更改 mysql 安装目录所属用户与用户组
cd /usr/local
sudo chown -R root:wheel mysql

# 执行 scripts 目录下的 mysql_install_db 脚本完成一些默认的初始化(创建默认配置文件、授权表等)
cd /usr/local/mysql
sudo scripts/mysql_install_db --user=mysql

最后显示OK后表示安装成功。测试启动、重启与停止：

cd /usr/local/mysql

# 启动
sudo support-files/mysql.server start

# 重启
sudo support-files/mysql.server restart

# 停止
sudo support-files/mysql.server stop

# 检查 MySQL 运行状态
sudo support-files/mysql.server status

还要初始一下 MySQL root 密码：

1. 先停止 MySQL server，用`sudo support-files/mysql.server stop`命令。
2. 打开Terminal,键入: sudo ./bin/mysqld_safe --skip-grant-tables
3. 打开另一个新的Terminal窗口：

＃ 进入mysql
sudo /usr/local/mysql/bin/mysql -u root

＃ 更新mysql密码，新旧版本命令不一样，这是5.7.9以上版本
 UPDATE mysql.user SET authentication_string=PASSWORD('新密码') WHERE User='root';

 ＃ flush privileges 命令本质上的作用是将当前user和privilige表中的用户信息/权限设置从mysql库(MySQL数据库的内置库)中提取到内存里。MySQL用户数据和权限有修改后，希望在"不重启MySQL服务"的情况下直接生效，那么就需要执行这个命令。通常是在修改ROOT帐号的设置后，怕重启后无法再登录进来，那么直接flush之后就可以看权限设置是否生效。而不必冒太大风险。
 FLUSH PRIVILEGES;

 # 退出
 \q

这个时候重启 MySQL 就可以：sudo support-files/mysql.server start

用以下命令进入：

cd /usr/local/mysql/bin
./mysql -u root -p
<你设置的mysql密码>

Mac下安装前端模板引擎Jinja2

Veröffentlicht am 2016-09-16   |  

廖大神的python教程对没有web开发经验的小白来说难度有点大啊，实战day1就把我拦下来了，Jinja2网上查下来都是使用easy_install或者pip安装。

sudo easy_install Jinja2
sudo pip install Jinja2

我遇到的问题是：

Searching for MarkupSafe
Reading https://pypi.python.org/simple/MarkupSafe/
error: ('The read operation timed out',)

然后发现还有一种方法是从tarball版本安装，其实就是把Jinja2安装包从官网下载下来、解包、安装。

步骤如下：

  1. 进入Jinja2官网下载“Jinja2-2.8.tar.gz”安装文件;

  2. 打开终端，输入：

    cd /Users/用户名/Downloads   （进入下载目录）
tar -xzvf Jinja2-2.8.tar.gz  (解压文件)
cd Jinja2-2.8               （进入目录）
sudo python setup.py install (安装Jinja2)

到这里发现还是碰到了问题Searching for MarkupSafe Reading https://pypi.python.org/simple/MarkupSafe/ error: ('The read operation timed out',)

一样的原因？突然灵机一动，于是进MarkupSafe官网下载“MarkupSafe-0.23.tar.gz”安装文件，按照上面的步骤再来一遍，终于出现了Finished processing dependencies for Jinja2==2.8文字，这算是安装成功了。
